## 1. Comment on les traite?

Ces rapports PVsyst VC0 sont la **pierre angulaire technique** de chaque projet. Voici le workflow recommandé :

**Ingestion dans l'outil kWh :** Chaque rapport devrait alimenter directement la fiche site/lead dans l'Optimizer. Concrètement, les données clés à extraire et stocker par site sont : puissance système (kWc), nombre de modules, modèle panneau + onduleur, production annuelle (Sim, P50, P90, P95), PR, productible (kWh/kWc), dégradation annuelle, configuration (inclinaison, azimut, type de montage), et la table de vieillissement 25 ans.

**Rôle dans le pipeline commercial :** Le VC0 arrive après la visite Rematek, donc il marque la transition du lead vers le stage "design_mandate_signed" ou au minimum "epc_proposal_sent". C'est le moment où tes chiffres estimatifs (calculateur ROI) sont remplacés par des **données validées par l'EPC**.

**Action immédiate :** Créer un champ `pvsystReportId` ou `vc0Data` dans le modèle site/lead pour stocker les données PVsyst structurées. Ça permet de basculer automatiquement la présentation client des estimations vers les chiffres réels.

## 2. Comment on améliore notre outil avec ça?

Plusieurs améliorations concrètes :

**A. Calibrer le CashflowEngine avec des données réelles.** Ton calculateur ROI utilise 1250 kWh/kWp d'irradiance. Rematek obtient 1026-1033 kWh/kWc de productible. C'est un écart significatif (~17%) qui s'explique par les pertes (encrassement neige ~27%, ombrages sheds, pertes thermiques, câblage, etc.). **Tu devrais utiliser ~1030 kWh/kWc comme productible net dans tes estimations**, pas 1250 d'irradiance brute. Sinon tu sur-estimes la production et le payback sera plus long que prévu.

**B. Ajouter les pertes d'encrassement neige.** Les deux rapports montrent 100% de perte en janvier-février et 70% en mars (neige sur toit plat). C'est un facteur majeur au Québec que ton outil n'intègre pas. Ajouter un profil de production mensuel serait un game changer pour la crédibilité.

**C. Intégrer un ratio DC:AC réaliste.** Rematek utilise 1.44-1.47. Ton outil devrait calculer le nombre d'onduleurs en fonction de ce ratio plutôt que d'assumer 1:1.

**D. Ajouter le gain bifacial.** Les deux projets utilisent des panneaux bifaciaux (facteur 80%). Le gain arrière est de 3.6-7.4% d'irradiance additionnelle selon l'albédo. C'est un paramètre que tu pourrais exposer dans la présentation client.

**E. Valider ta dégradation.** Tu utilises 0.5%/an dans le calculateur, Rematek utilise 0.4%/an. Ça change le cumul 25 ans. Je recommande d'aligner sur 0.4% puisque c'est ce que l'EPC utilise dans ses simulations.

## 3. Est-ce possible de produire un rapport comme celui-ci?

**Réponse courte : Non, et tu ne devrais pas essayer.**

Un rapport PVsyst VC0 nécessite :
- Le logiciel PVsyst (licence ~2000$/an) avec base de données météo Meteonorm
- Une visite physique du site (toiture, orientation, ombrages, structure)
- L'expertise d'un ingénieur pour configurer les sheds, l'espacement, les strings, le choix onduleur
- La modélisation bifaciale avec albédo calibré
- L'évaluation P50-P90-P95 avec incertitudes

C'est le travail de Rematek en tant qu'EPC. Tenter de le reproduire diluerait ta valeur (tu es l'optimiseur/courtier, pas l'EPC) et engagerait ta responsabilité sur des chiffres de production.

**Ce que tu PEUX faire :** Produire un **rapport de pré-faisabilité kWh** qui se positionne EN AMONT du VC0. Ton rapport dirait : "Selon notre estimation préliminaire, ce site pourrait supporter ~2700 kWc et produire ~2800 MWh/an. Voici le potentiel financier. Pour des chiffres certifiés, l'étape suivante est le design mandate avec notre EPC partenaire Rematek."

C'est exactement ce que fait déjà ta page Présentation. Tu pourrais ajouter une section "Comparaison estimé vs VC0" une fois le rapport Rematek reçu, pour montrer la cohérence de tes estimations initiales.
