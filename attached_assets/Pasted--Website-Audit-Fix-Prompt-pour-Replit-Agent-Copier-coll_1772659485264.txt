**Fichier:** `client/src/pages/landing.tsx`

- Ajouter formatage automatique au blur: `(514) 427-8871`
- Si le téléphone est fourni (optionnel), il DOIT être valide
- Ajouter message d'erreur i18n: "Numéro de téléphone invalide" / "Invalid phone number"

### FIX-03: autorisation-hq.tsx — Sécurité localStorage

**Fichier:** `client/src/pages/autorisation-hq.tsx`

- Remplacer TOUS les `localStorage.setItem` et `localStorage.getItem` par `sessionStorage`
- Les données de compte HQ ne doivent PAS persister après fermeture du navigateur
- Vérifier qu'aucune donnée sensible (numéro de compte, adresse) ne reste dans localStorage

### FIX-04: Toasts error/success sur formulaires publics

**Fichiers:** `client/src/pages/landing.tsx`, `client/src/pages/analyse-detaillee.tsx`, `client/src/pages/autorisation-hq.tsx`, `client/src/pages/sign-agreement.tsx`

Pour chaque formulaire:
- Ajouter `import { useToast } from "@/hooks/use-toast"` (déjà dans le projet)
- Sur soumission réussie: toast success avec message i18n ("Formulaire soumis avec succès" / "Form submitted successfully")

- Ajouter `disabled={isSubmitting}` sur le bouton submit pour éviter double-soumission
- Ajouter un spinner/loading state pendant la soumission

### FIX-05: analyse-detaillee.tsx — Validation compte HQ

**Fichier:** `client/src/pages/analyse-detaillee.tsx`

- Ajouter validation regex pour numéro de compte HQ (format attendu: chiffres, 10-12 caractères)
