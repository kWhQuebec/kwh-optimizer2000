## CONTEXTE

Un audit expert du site web public kWh Optimizer a identifié un score de **6.5/10** avec des faiblesses majeures en design (top 25-30%, pas top 1%), navigation (redondance Ressources/Nouvelles), SEO (sitemap incomplet), et images (déficientes). Ce prompt combine les correctifs techniques + un redesign visuel complet pour passer à **8.5+/10**.

Le site est React 18 + Wouter + TanStack Query + Radix UI + Tailwind CSS. On utilise déjà shadcn/ui, Framer Motion, et lucide-react — PAS de packages tiers à installer.

**RÈGLES ABSOLUES:**
1. Ne JAMAIS casser les 548 tests existants. `npm test` après chaque groupe de changements.
2. Tout le contenu public = bilingue (FR/EN) via `client/src/lib/i18n.tsx`.
3. Ne PAS modifier les pages protégées `/app/*` sauf les 3 bugs listés en Phase 5.
4. Utiliser Framer Motion (déjà installé) pour toutes les animations.
5. Chaque composant modifié doit rester accessible (ARIA labels, keyboard nav, contrast ratio AA minimum).
6. NE PAS installer de nouveaux packages npm sauf si absolument nécessaire. Utiliser ce qui existe déjà (shadcn/ui, Radix UI, Tailwind, Framer Motion, lucide-react, Zod, react-hook-form).

**DIRECTION DESIGN:**
- **Style:** Clean Energy / SaaS Enterprise — professionnel, aspirationnel, trustworthy
- **Palette:** Bleu profond (#0F172A slate-900) + accents énergie (ambre/gold #F59E0B). Remplacer le orange générique actuel.
- **Typo:** Plus Jakarta Sans (headings 700/800) + Inter (body 400/500) — importer via Google Fonts
- **Mood:** Un directeur d'immeuble C&I doit sentir qu'il parle à une firme d'ingénierie top-tier, pas une startup
- **Animations:** Subtiles, professionnelles (Framer Motion). Pas de gimmicks.

## PHASE 4 — Formulaires (Qualité production)

### FORM-01: Validation Zod sur sign-agreement.tsx

Ajouter un schéma Zod:
- `signerName` — requis, min 2 chars
- `signerEmail` — requis, format email
- `signerTitle` — requis, min 2 chars
- `signature` — requis, non vide
- `agreedToTerms` — requis, `true`

Utiliser `react-hook-form` + `zodResolver`. Messages d'erreur i18n.

### FORM-02: Validation téléphone sur landing.tsx

Regex téléphone canadien: `/^(\+1)?[\s.-]?\(?\d{3}\)?[\s.-]?\d{3}[\s.-]?\d{4}$/`
Format auto au blur: `(514) 427-8871`. Message erreur i18n.

### FORM-03: sessionStorage sur autorisation-hq.tsx

Remplacer TOUS les `localStorage` par `sessionStorage`. Données HQ ne doivent PAS persister.

### FORM-04: Toasts success/error sur TOUS les formulaires publics

Fichiers: `landing.tsx`, `analyse-detaillee.tsx`, `autorisation-hq.tsx`, `sign-agreement.tsx`
- Toast success bilingue après soumission réussie
- Toast error bilingue sur erreur API
- `disabled={isSubmitting}` sur les boutons submit
- Spinner/loading state pendant la soumission

### FORM-05: Champs requis marqués

Sur tous les formulaires: astérisque rouge `*` sur les labels requis. Champs optionnels: "(optionnel)" / "(optional)".
